Create a wide 16:9 fictional rehearsal planning artifact titled visually only in small text as "Use Case: Rehearsal Blocking Atlas". Show how GPT Image 2 can turn a plain-text eight-count rehearsal note into a precise stage-floor blocking atlas.

Scene/backdrop: a clean black-box theater planning wall, not a product photo and not a tabletop apparatus. The image should feel like a finished rehearsal reference pinned flat to a wall: matte paper, crisp vector-like floor maps, colored path ribbons, small count bars, and side notes.

Main visual structure: one large overhead stage-floor map dominating the center, with upstage/downstage and stage-left/stage-right labels; four abstract performer tokens named A, B, C, D; translucent path ribbons with arrows across counts 1-8; entrance/exit gates; hold markers; facing triangles; spacing-safe zones. Along the left edge, a small input note strip with invented shorthand: "A enters USL on 1, B crosses CS on 3, C holds DSL 4-5, D exits USR on 8". Along the right edge, a review rail with check chips: COUNT LOCK, PATH CLEAR, HOLD VISIBLE, FACING SET, EXIT CLEAN.

Style: sophisticated rehearsal-document design, crisp editorial diagram, restrained but varied colors, high legibility, no photoreal dancers. Use abstract tokens and arrows only, no bodies or faces.

Prompt mechanism shown: text-to-spatial-plan, count alignment, path continuity, collision check, reusable rehearsal reference.

Safety and originality constraints: no real dancers, no realistic faces, no celebrity likeness, no minors, no real troupe or venue names, no real songs, no cultural dance costumes or named dance traditions, no brands/logos, no copyrighted choreography, no Labanotation symbols, no copied notation diagrams, no politics, no explicit content, no violence, no macro object photography, no tray/controller/lab device, no 2x2 or 2x3 research board.
