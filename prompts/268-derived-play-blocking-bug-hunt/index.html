Full Prompt

Create a wide 16:9 playable puzzle sheet titled in small text "Derived Play: Blocking Bug Hunt". It is derived from the Rehearsal Blocking Atlas: players inspect a fictional stage-floor plan generated from a count sheet and repair movement-continuity errors.

Scene/backdrop: flat printed puzzle poster on a neutral rehearsal-room wall, crisp and legible. No product photo, no tabletop game pieces, no physical apparatus.

Main layout: one large overhead stage map on the left with abstract performer tokens A, B, C, D and colored path ribbons. Intentionally mark five safe fictional errors with numbered warning badges: 1 LATE ENTRANCE, 2 PATH COLLISION, 3 WRONG FACING, 4 MISSED HOLD, 5 OFF-COUNT EXIT. On the right, a vertical repair loop with five steps: SPOT, NAME, PATCH, RECOUNT, PASS. Along the bottom, correction cards showing tiny before/after mini-maps for each error, with checkboxes and count numbers.

Visual details: use clean stage notation language: upstage/downstage/stage left/stage right, count ticks 1-8, dashed original paths, solid repaired paths, facing triangles, hold pause marks, spacing halos. Make it clear this is a game about checking visual generation consistency, not a real dance instruction.

Style: polished editorial diagram, playful but professional, varied palette, high contrast, readable labels, compact layout that does not resemble a 2x2/2x3 contact sheet.

Prompt mechanism shown: consistency QA, spatial reasoning, beat alignment, rule-based visual repair, before/after edits.

Safety and originality constraints: no real dancers, no human faces or bodies, no celebrity likeness, no minors, no cultural dance traditions, no real songs, no real venues/troupes, no brands/logos, no copyrighted choreography, no copied notation symbols, no politics, no explicit content, no violence, no medical/legal claims, no macro object photography, no board game table, no route/transit map metaphor.
